How it works

The merchant graph
you already have

Knot's existing infrastructure sees which merchants each cardholder is linked to. MerchantMatch™ layers a recommendation engine on top — surfacing the right merchant at the right moment, inside the issuer's existing app.

01
Read the graph
Knot already holds a unique dataset: which merchants each cardholder is linked to across the network. No new data collection needed — the signal is already there.
02
Find the pattern
Cardholders with Spotify + Netflix + Uber have an 84% overlap with Airbnb. That pattern lives in the graph. MerchantMatch™ surfaces it as a personalised recommendation.
03
One-tap link
The issuer surfaces the recommendation in their app. The cardholder links their card with one tap via Knot's existing CardSwitcher™ flow. Zero new infrastructure required.

What's inside

Built on infrastructure
Knot already owns

🔗
Merchant graph engine
Aggregates anonymised link data across Knot's network to surface statistically significant merchant co-occurrence patterns. No PII required.
🎯
Issuer-branded recommendations
Recommendations surface inside the issuer's existing mobile app — fully white-labelled. The cardholder sees their bank recommending it, not Knot.
⚡
One-tap via CardSwitcher™
Linking is handled by Knot's existing CardSwitcher™ flow. No new UX, no new SDK — MerchantMatch™ is a logic layer on top of what already works.
📊
Issuer analytics dashboard
Issuers see card presence scores, match rates, and incremental merchant links over time. Turns a utility into a trackable growth metric.
🔒
Privacy-first by design
Patterns derived from aggregate behaviour, not individual profiling. Fits cleanly within PSD2/GDPR constraints — a key requirement for European deployment.
🌍
EU-ready from day one
The merchant graph is global. Southern European issuers (SIBS, Bizum-linked banks, BancoBPM) can activate immediately — no regional merchant negotiation needed.
Business model

Three ways Knot earns

MerchantMatch™ is additive to Knot's existing revenue — every recommendation that converts is a CardSwitcher™ event, which Knot already monetises. The model layers on top.

Model 01
Per-link fee
Issuer pays per successful merchant link generated via a MerchantMatch™ recommendation. Aligned incentives — Knot earns when it delivers.
$0.25–$0.75 / successful link
stacks on existing CardSwitcher™ fee
Model 02
SaaS dashboard
Monthly platform fee for access to the analytics dashboard, graph insights, and recommendation tuning. Sells to the CMO, not just the CTO.
$2,000–$15,000 / month
based on cardholder volume
Model 03
Merchant sponsorship
Merchants pay to be recommended to high-fit cardholder segments. Clean, performance-based — only charged on confirmed links, not impressions.
CPA model · $1–$4 / link
opt-in for Knot merchant network
