API Reference

All endpoints use HTTP Basic Auth with your client_id (username) and secret (password). Pass Knot-Version: 2.0 on every request. Choose your environment from the base URLs below.

Authentication
Credentials differ between development and production environments. Retrieve them from your Knot Customer Dashboard.

Errors
Knot uses standard HTTP status codes. SDK errors are encapsulated in a KnotError object.
CodeMeaning
INVALID_SESSIONSession ID is malformed or doesn't exist.
EXPIRED_SESSIONSession has expired. Create a new one via /session/create.
INVALID_CLIENT_IDCredentials are incorrect or for the wrong environment.
INTERNAL_ERRORUnexpected server error. Contact Knot support with the task_id.
Create Session CardSwitcher™
Creates a session used to initialise the Knot SDK. Generate a new session on every SDK initialisation — do not reuse session IDs.
POST /session/create Authenticated · Basic Auth
Request body
ParameterTypeDescription
external_user_id string Required Your internal user identifier. Returned in all downstream webhooks for this session.
type string Required card_switcher or transaction_link
merchant_ids int[] Optional Restrict the SDK to specific merchants. A single ID sends the user directly to that merchant's login.
entry_point string Optional Label the entry point (e.g. "onboarding", "push-notif"). Returned in the AUTHENTICATED webhook for funnel analytics.
metadata object Optional Pass-through fields (e.g. reference_token, trace_id) returned in webhooks.
Example request
cURL
curl -X POST 'https://development.knotapi.com/session/create' \
  -u '[client_id]:[secret]' \
  -H 'Content-Type: application/json' \
  -H 'Knot-Version: 2.0' \
  -d '{
    "external_user_id": "user_3dcbb19a",
    "type": "card_switcher",
    "entry_point": "onboarding"
  }'
SDK initialisation (JavaScript)
JavaScript
import KnotapiJS from "knotapi-js";
const knotapi = new KnotapiJS();

knotapi.open({
  sessionId:   "sess_fb5aa994",  // from /session/create
  clientId:    "client_67890",
  environment: "development",
  merchantIds: [17],             // optional: restrict to one merchant
  entryPoint:  "onboarding",
  useCategories: true,
  useSearch:     true,
  onSuccess: (details) => console.log("Switched", details),
  onError:   (code, msg) => console.error(code, msg),
  onEvent:   (event, merchant, payload, taskId) => {},
  onExit:    () => {}
});
Response

JSON

  "expires_at": 1741968000,
  "environment": "development"
}
List Merchants CardSwitcher™
Returns the full merchant list for a given product type. Providing an external_user_id adds per-user connection status to each merchant.
GET /merchants
Query parameters
ParameterTypeDescription
type string Required card_switcher or transaction_link
external_user_id string Optional Adds a status field per merchant reflecting the user's lifecycle state.
platform string Optional Filter by ios, android, or web.
search string Optional Keyword search — use to power an in-app merchant search experience.
Response
✓ 200 OK
JSON
{
  "merchants": [
    {
      "id": 11,
      "name": "Uber",
      "logo_url": "https://cdn.knotapi.com/merchants/11.png",
      "categories": ["Ride sharing", "Delivery"],
      "platforms": ["ios", "android", "web"],
      "status": "linked"  // only when external_user_id is provided
    }
  ]
}
Switch Card CardSwitcher™
Send card data to Knot immediately after receiving an AUTHENTICATED webhook with send_card: true.
Timing: Call this endpoint within 15 seconds of the AUTHENTICATED webhook. The window closes after that.
POST /card/switch Also available: /card/switch/jwe (JWE-encrypted)
Request body
JSON
{
  "task_id": 25605,          // from AUTHENTICATED webhook
  "card": {
    "number":    "4111111111111111",
    "exp_month": 12,
    "exp_year":  2027,
    "cvv":       "123",
    "name":      "Rita Neves"
  }
}
Get Recommendations MerchantMatch™ new
Returns a ranked list of merchants the cardholder is likely to link, based on co-occurrence patterns in Knot's merchant graph. No PII is used — all signals are derived from aggregate, anonymised behaviour across the network.
POST /merchantmatch/recommendations
Request body
ParameterTypeDescription
external_user_id string Required The cardholder to generate recommendations for. Must have at least one linked merchant.
limit integer Optional Max recommendations to return. Default: 5. Max: 20.
exclude_merchant_ids int[] Optional IDs to exclude (e.g. already linked merchants, issuer-suppressed merchants).
platform string Optional Filter recommendations to merchants available on a specific platform.
Example request
cURL
curl -X POST 'https://production.knotapi.com/merchantmatch/recommendations' \
  -u '[client_id]:[secret]' \
  -H 'Content-Type: application/json' \
  -H 'Knot-Version: 2.0' \
  -d '{
    "external_user_id": "user_3dcbb19a",
    "limit": 5,
    "platform": "ios"
  }'

Query Merchant Graph MerchantMatch™ new
Directly query merchant co-occurrence data. Useful for building custom recommendation logic or powering your own analytics UI.
POST /merchantmatch/graph/query
Request body
ParameterTypeDescription
seed_merchant_ids int[] Required Starting merchants to compute co-occurrence from.
limit integer Optional Number of results to return. Default: 10.
Response
✓ 200 OK
JSON
{
  "seed_merchant_ids": [11, 17, 3],  // Uber, Spotify, Netflix
  "co_occurrences": [
    {
      "merchant": { "id": 44, "name": "Airbnb" },
      "overlap_rate": 0.88,   // fraction of seed-holders who also have this
      "network_rank": 1
    }
  ]
}
Record Link Event MerchantMatch™ new
Record when a cardholder successfully links their card via a MerchantMatch™ recommendation. Used for attribution, analytics, and improving graph accuracy over time.
POST /merchantmatch/events/link
Request body
JSON
{
  "external_user_id": "user_3dcbb19a",
  "merchant_id": 44,
  "recommendation_id": "rec_8f3a2b1c",  // from /recommendations response
  "source": "merchantmatch_recommendation"
}
Webhook Events
Knot sends POST requests to your webhook URL from IP 35.232.249.218. Respond with 200 within 10 seconds — Knot retries up to 2× on failure.
CardSwitcher™ lifecycle
1
User opens SDK, selects merchant → OPENED
2
User submits credentials → CREDENTIALS_RECEIVED. If wrong → CREDENTIALS_FAILED
3
MFA needed → OTP_REQUIRED. Auth succeeds → AUTHENTICATED with send_card: true
4
Call POST /card/switch within 15 seconds → CARD_SWITCHED
AUTHENTICATED payload
JSON · Webhook payload
{
  "event":            "AUTHENTICATED",
  "session_id":       "sess_fb5aa994-ed1c-4c3e-b29a-b2a53222e584",
  "task_id":          25605,
  "external_user_id": "user_3dcbb19a-b2f1-4a7b-8792-d76027b627b3",
  "merchant": { "id": 11, "name": "Uber" },
  "data": {
    "card_id":    "123456789",
    "send_card":  true,
    "entrypoint": "onboarding"
  },
  "timestamp": 1710864923198
}
💡 The MerchantMatch™ loop

Recommend → Switch → Record

Call /merchantmatch/recommendations to find what to link → open a session with /session/create pointing to that merchant → execute the switch with /card/switch → confirm attribution with /merchantmatch/events/link. Same auth, same SDK, same webhooks — MerchantMatch™ is purely additive.
