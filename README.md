## Why MerchantMatch™?

Knot's core product creates a unique data asset: a real-time graph of which merchants cardholders actively use. Most of that signal goes untapped after the switch event. MerchantMatch™ closes that loop by:

- Surfacing co-occurrence patterns across the merchant graph
- Serving ranked recommendations at the moment of highest intent (right after a switch)
- Creating a new monetization layer for issuers and a retention tool for cardholders

The API reference was designed to be consistent with Knot's existing CardSwitcher™ conventions — same auth model, same versioning header, same webhook lifecycle.
